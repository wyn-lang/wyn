# Wyn Package Manager

The Wyn package manager handles project creation, dependency management, and builds.

## Quick Start

```bash
# Create a new project
make new NAME=myapp
cd myapp

# Or initialize in existing directory
make init

# Build your project
make stage0
./build/stage0 -o build/myapp src/main.wyn
```

## Project Structure

A Wyn project has this structure:

```
myapp/
├── wyn.toml          # Project manifest
├── wyn.lock          # Dependency lock file (auto-generated)
├── src/
│   └── main.wyn      # Entry point
├── build/            # Build output
└── .wyn/
    └── deps/         # Downloaded dependencies
```

## Manifest (wyn.toml)

```toml
[package]
name = "myapp"
version = "0.1.0"
description = "My awesome app"
license = "MIT"

[build]
entry = "src/main.wyn"

[dependencies]
github.com/user/repo = "^1.0.0"
```

## Dependencies

### Adding Dependencies

Dependencies are specified in `wyn.toml`:

```toml
[dependencies]
# GitHub packages
github.com/user/repo = "^1.0.0"
github.com/user/repo = "v2.1.0"

# Local packages
./libs/mylib = "*"

# Any version
some-package = "*"
```

### Version Constraints

| Constraint | Meaning |
|------------|---------|
| `*` | Any version |
| `1.2.3` | Exact version |
| `^1.2.3` | Compatible (same major, >= 1.2.3) |
| `~1.2.3` | Approximately (same major.minor, >= 1.2.3) |
| `>=1.0.0` | Greater than or equal |
| `>1.0.0` | Greater than |
| `<=1.0.0` | Less than or equal |
| `<1.0.0` | Less than |

### Import Syntax

```wyn
# Standard library
import math
import http

# Relative imports
import ./utils
import ../shared/config

# External packages
import github.com/user/package
import github.com/user/package/submodule

# Aliasing
import http as h
import github.com/user/long_package_name as pkg

# Selective imports
import math { sin, cos, PI }
```

## Lock File (wyn.lock)

The lock file records exact versions of all dependencies:

```toml
# This file is auto-generated by wyn
# Do not edit manually

[[package]]
name = "github.com/user/repo"
version = "1.2.3"
source = ".wyn/deps/github.com/user/repo"
```

## Package Resolution

1. Standard library: `import http` → built-in
2. Relative path: `import ./foo` → `./foo.wyn` or `./foo/` folder
3. Package: `import github.com/x/y` → downloaded to `.wyn/deps/`

When the full `wyn` CLI is available:

```bash

wyn add <package>   # Add dependency
wyn remove <package># Remove dependency
wyn update          # Update dependencies
wyn build           # Build project
wyn run             # Build and run
wyn test            # Run tests
wyn clean           # Remove build artifacts
```

The package manager is implemented in Wyn (`src/pkg/`) and will be fully functional once Stage 1 self-hosting is complete. Currently available via Makefile:

- `make new NAME=x` - Create new project
- `make init` - Initialize project
- `make clean` - Clean build artifacts
