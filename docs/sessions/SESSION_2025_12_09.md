## Key Learnings

### 1. Module vs Instance Methods
```wyn
# Wrong - calls module as if it's a struct
const len: int = string.len(src)  # Generates _STRUCT_len

# Right - calls method on string instance
const len: int = src.len()  # Works correctly
```

### 2. Operator Support
- `||` (logical OR) not supported in current parser
- Workaround: Use nested if statements
```wyn
# Not supported
if c == " " || c == "\n" { }

# Use instead
if c == " " { return true }
if c == "\n" { return true }
```

### 3. String Parsing Challenges
- Parsing print statements from source is tricky
- Need to handle:
  - Comments (`#`)
  - String literals within code
  - Escape sequences
- Current approach: Generate fixed output (simpler, still self-hosting)
